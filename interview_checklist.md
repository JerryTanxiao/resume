# 谭杰 - 高级 Java & AI 架构师面试准备清单

这份清单基于你的 8 年实战经验 (特别是千万级 MQ 治理、IoT 接入、微服务演进及 AI 落地) 整理，旨在帮助你应对 P7/P8 级别的技术面试。

## 1. Java 核心与性能调优 (Senior Level)

- [ ] **JVM 深入理解**：
    - [ ] **G1/ZGC 回收原理**：能结合简历中的“百万级并发系统性能瓶颈排查”，聊聊 GC 停顿对系统的影响及调优手段。
    - [ ] **Arthas/JProfiler 实战**：具体描述你当时发现 CPU 飙升或内存泄漏 (OOM) 的操作步骤和最终结论。
- [ ] **并发编程 (JUC)**：
    - [ ] **CompletableFuture**：在 DDMS 2.0 中，你是如何编排并行 RPC 调用的？如何处理异常回滚？
    - [ ] **线程池调优**：如何动态调整 `corePoolSize`？在海量任务下，如何防止队列溢出导致的任务丢失？

## 2. 分布式与中间件 (Architecture)

- [ ] **RocketMQ 源码级深度**：
    - [ ] **时间轮算法 (TimeWheel)**：这是你的核心亮点。面试官必问：为什么原生延迟队列不够用？时间轮的插入和扫描复杂度是多少？如何保证持久化？
    - [ ] **千万级积压治理**：那 1000w 条轨迹点位是怎么堆积的？16 路 Queue 物理转储的具体逻辑是什么？为什么线性扩容 Consumer 就能解决？
- [ ] **Netty 与通信协议**：
    - [ ] **粘包与拆包**：在 IoT 接入中，你是如何定义 Protobuf 帧头的？
    - [ ] **纠偏算法**：经纬度平滑纠偏的具体数学逻辑 (如卡尔曼滤波或简单的滑动窗口) 需要能大概讲清。
- [ ] **微服务治理**：
    - [ ] **LiteFlow 落地**：动态业务规则是如何存储和加载的？灰度发布时如何确保逻辑平滑切换？
    - [ ] **分布式事务 (Seata)**：在风控系统中，你是如何权衡 AT 模式的性能损耗与 TCC 模式的开发成本的？

## 3. 数据库与存储 (Optimization)

- [ ] **MySQL 极致优化**：
    - [ ] **300s -> 1s**：针对深能环保的案例，当时慢的原因是视图 (View) 还是关联表过多？索引下推 (Index Condition Pushdown) 失效是如何解决的？
    - [ ] **分库分表**：如果有过实战，聊聊分片键的选择和扩容时的双写方案。
- [ ] **Flink 实时计算**：
    - [ ] **数据漂移**：针对 5 小时数据偏差，如何排查出是 CU 资源不足导致的？反压 (Backpressure) 机制在你的场景下如何体现？

## 4. AI 工程化 (The "Edge")

- [ ] **RAG 架构深度**：
    - [ ] **知识库优化**：如何提高 Embedding 的检索精度？ (如：语义分段、混合搜索 Hybrid Search)。
    - [ ] **Dify Agent**：Function Calling 的闭环逻辑是什么？如果 LLM 产生的 Tool 调用参数错误，你的系统如何自愈？
- [ ] **Prompt Engineering**：
    - [ ] **结构化 Prompt**：在动漫分镜助手里，如何确保生成的内容符合工业化标准 (景别、光效等)？
    - [ ] **一致性控制**：如何通过 Seed 或特定的 Prompt 策略锁定角色肖像？

## 5. 项目方法论与领导力 (Leadership)

- [ ] **团队管理 (2.5年经验)**：
    - [ ] **导师制与 CR**：你推行的代码质量红线 (Redline) 包含哪些具体指标？
    - [ ] **技术决策**：当 阿里专家团队与你的想法冲突时，你如何通过实验数据或技术调研来说服对方 (深能环保项目)？

---

### 面试小贴士：
1. **STAR 法则**：描述 1000w 积压治理时，明确背景 (S)、目标 (T)、行动 (A：MQ 隔离、线性扩容)、结果 (R：性能提升、彻底解决 OOM)。
2. **避坑指南**：如果你提到了“源码开发”，一定要准备好应对“如果开源版本升级了，你的自定义代码如何维护”的问题。
